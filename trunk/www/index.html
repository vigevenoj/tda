Goal is to provide a small utility helping offline analysis of production enviroments like application servers.

Background

There is a quite nice Thread Dump Analysis Tool for Websphere environments but which only works with IBM JDKs. Motivation for this little tool was to have something similar for Sun JDKs. See JavaPedia: ThreadDump for links.

Project Summary

The TDA Thread Dump Analyzer for Java is a small Swing GUI for analyzing Thread Dumps and Heap Information generated by the Sun Java VM (currently tested with 1.4.x releases). It parses the thread dumps and class histogram from a provided log file. TDA needs at least Java 1.4.x to run.

It provides statistic informations about the found thread dumps, gives information about locked monitors and waiting threads. It can diff between threads to look for similarities and has a "long-running-thread" detection. It also provides overview of heap objects at a thread dump, if class histogram was logged.

Features are in detail:

  • parsing all thread dumps of a log file, using a specified regular expression for log file time stamps (thread dumps don't have any date information by default), displaying them in a tree.
  • Display of statistic data about the found thread dumps, display of monitors, waiting, sleeping and locking threads,
  • integrated class histogram analysis for a specific thread dump (using the -XX:+PrintClassHistogram option of the VM), can also be parsed from a second file, if loggc-Option was used (to log garbage collections to a different log file).
  • parser interface for integrating dump parser for other VMs
  • displays deadlock information from the thread dump.

Features planned:

  • Storing of current session, to not to have to load all the log files again.
  • Better Dump navigation with context sensitive dump information.
  • More Parsers, including jmap (from JDK 1.5)
  • request thread dumps remotely using jmx (for JDK's >= 1.5) in progress.
  • Add additional logfiles (useful if logfiles are rotated) done for next release.
  • If possible provide plugins for Eclipse and Netbeans.
  • Parse hprof-file generated by -XX:+HeapDumpOnOutOfMemoryError Option in VM.

The emphasis of the TDA is to provide analysis of offline information for production environment to get a first idea what is happening on the system, if it has problems. Then do further analysis with projects like the Glassbox-Inspector if you need to analyse the productions environment with low impact on the performance and load or with conventional tools if you can analyse the problem elsewhere.

In addition to TDA you might find the GCViewer helpful. It parses the output of the garbage collector from a loggc file and displays it graphically, so you can analyse the memory usage over the time.

TDA includes a short tutorial showing how to use the tool.

Download

You can download the 1.0 Final release in the files section. Look at the tutorial in the help menu for usage information. Start scripts are very basic at the moment, look inside them, you might need to adjust the memory settings if you want to parse very huge log files (>250MB).
