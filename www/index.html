Mission Statement

Provide a small utility helping offline analysis of production enviroments like application servers.


Background

There is a quite nice Thread Dump Analysis Tool for Websphere environments but which only works with IBM JDKs. Motivation for this little tool was to have something similar for Sun JDKs


Project Summary

The TDA Thread Dump Analyzer is a small Swing GUI for analyzing Thread Dumps generated by the SUN JVM (currently tested with 1.4.x releases). It parses the thread dumps from a provided log file.

It provides statistics informations about the found thread dumps, gives information about locked monitors and waiting threads. It can diff between threads to look for similarities and has a "long-running-thread" detection.

Features are in detail:

  • parsing all thread dumps of a log file, using a specified regular expression for log file time stamps (thread dumps don't have any date information by default), displaying them in a tree.
  • Display of statistic data about the found thread dumps, display of monitors, waiting, sleeping and locking threads,
  • integrated class histogram analysis for a specific thread dump (using the -XX:+PrintClassHistogram option of the VM)
  • parser interface for integrating dump parser for other VMs
  • displays deadlock information.
  • includes heap information for a thread dump (if parameter -XX:+PrintClassHistogram was included), can also be parsed from a second file, if loggc-Option was used (to log to a different VM)
Features planned:
  • Storing of current session, to not to have to load all the log files again.
  • Better Dump navigation with context sensitive dump information.
  • More Parsers, including jmap (from JDK 1.5)
  • Online request of a thread dump from a VM.

The emphasis of the TDA is to provide analysis of offline information for production environment to get a first idea what is happening on the system, if it has problems and then do further analysis with projects like the Glassbox-Inspector. In addition to TDA you might find the GCViewer helpful. It parses the about of the garbage collector from a loggc file and displays it graphically.

TDA includes a short tutorial showing how to use the tool.

Download

You can download a first beta release in the files section. Look at the tutorial in the help menu for usage information. Start scripts are very basic at the moment, look inside them, you might need to adjust the memory settings.
